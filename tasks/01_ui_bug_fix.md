# タスク01：言語切り替え時のウィンドウサイズバグ修正指示書

## 1. 目的
ユーザーがメニューの「表示 ➔ 言語」を切り替えた際、ウィンドウ全体のサイズが勝手に極端に小さく縮小してしまうUIバグを修正し、教育現場でストレスなく学べる環境を作る。

## 2. 着手手順・手がかり
1. **バックアップの作成**: 
   修正前に必ず現在の `src/aigis/ui/` などの対象フォルダを、消さずに `src/aigis/ui_backup_ui_bug_fix/` などの別名フォルダに変更・保存すること。
2. **コードの修正**:
   `MainFrame.java` の `rebuildUI()` 実行直前に、現在のウィンドウの `Dimension size = this.getSize();` および `Point location = this.getLocation();` を取得。
   UIの再構築（`pack()` 等）が走った直後に、`this.setSize(size);` および `this.setLocation(location);` を適用して、元のサイズと位置を強制維持させる。
3. **描画の整合性**:
   サイズ維持の前後で `revalidate()` と `repaint()` が正しく呼ばれ、文字だけが滑らかに切り替わるようにする。

## 3. 完了条件
- 言語切り替えを行っても画面サイズが完全に維持されること。
- 修正前のバックアップフォルダが正しくルートに保存されていること。
- ビルドが正常に通り、コードの差分（`git diff`）が提示されていること。
