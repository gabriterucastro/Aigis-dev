【3】タスク01：言語切り替え時のウィンドウサイズバグ修正
----------------------------------------------------------------

■ 対象ファイル：
  src/aigis/ui/MainFrame.java

■ 問題：
  メニューから言語（日本語 ⇔ English）を切り替えると、
  rebuildUI() 内で新フレームが固定サイズ 1400x1000 で生成され、
  位置も画面中央にリセットされてしまう。

■ 修正内容（L95-103）：
  【修正前】
    public void rebuildUI() {
        SwingUtilities.invokeLater(() -> {
            dispose();
            MainFrame frame = new MainFrame();
            frame.setSize(1400, 1000);          // 固定値
            frame.setLocationRelativeTo(null);  // 位置リセット
            frame.setVisible(true);
        });
    }

  【修正後】
    public void rebuildUI() {
        Dimension size = this.getSize();        // 現在サイズを退避
        Point location = this.getLocation();    // 現在位置を退避
        SwingUtilities.invokeLater(() -> {
            dispose();
            MainFrame frame = new MainFrame();
            frame.setSize(size);                // サイズ強制維持
            frame.setLocation(location);        // 位置強制維持
            frame.setVisible(true);
            frame.revalidate();
            frame.repaint();
        });
    }

■ バックアップ：src/aigis/backup_ui_080622_1826/

【5】タスク03：ズーム連動型2Dスケールバー表示機能の実装
----------------------------------------------------------------

■ 新規作成ファイル：
  src/aigis/gl/ScaleBar.java

■ 変更ファイル：
  src/aigis/gl/Renderer.java

■ 機能概要：
  カメラのDistanceとFOVの値をリアルタイムに参照し、
  3D座標→2D画面ピクセル変換を行って、画面左下に
  「地図の縮尺のようなスケールバー」をオーバーレイ表示する。

■ 計算ロジック（透視投影）：
  可視高さ(km)   = 2 × camera.distance × tan(camera.fov / 2)
  m/ピクセル     = 可視高さ(m) / viewportの高さ(px)
  目標バー長(m)  = m/ピクセル × viewportの幅 / 5
  表示値         = 1,2,5,10,20,50,100,200,500,1000m から最も近い値に丸め
  バー幅(px)     = 表示値(m) / m/ピクセル

■ 表示仕様：
  - 位置：左下隅（常時表示、モデル読み込み後から有効）
  - ラベル：「100 m」「1 km」など自動フォーマット
  - 白色の矩形バー＋黒縁取り＋両端ティックマーク
  - ズームイン・ズームアウトに連動してリアルタイム変化

■ Renderer.java への追加（renderOthers メソッド内）：
  private ScaleBar scaleBar = new ScaleBar();  // フィールド追加
  scaleBar.draw(gl, viewport[2], viewport[3], camera);  // 呼び出し追加
