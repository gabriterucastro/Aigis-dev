# AiGIS 拡張開発プロジェクト：メイン共通指令書

本ドキュメントは、Java/Swingアプリケーションである「AiGIS」の課題を解決し、最終発表で実際に動作するシステムを披露するための全体共通指令書である。

---

## ■ バックアップに関する絶対防衛線
- 開発プロセスを進める際は、必ず変更対象のフォルダを削除せず完全にコピー退避して保存せよ。
- 🚫 **退避先は必ずビルド対象ツリー（`src/src/` 配下）の外、すなわち `src/backups_disabled/` とすること。** `src/src/` 配下に `*_backup_*` 等を作ると、`make.sh` の `find ./src -name "*.java"` が同一 `package` のバックアップまでコンパイルし、「重複クラス」エラーでビルドが全滅する。
- 例：`src/aigis/ui/` を変更する場合 → `src/backups_disabled/ui_backup_20260622/` へコピーしてから編集する。

## ■ Agentの役割定義（Skillsベース）

実行するタスクに応じて、以下の3つの「スキル（役割）」を自己選択して切り替え、対応せよ。

### 1. 【UI/UX 改善Agent】（Skill: UIレイアウト・国際化バグ修正）
- **ミッション**: 言語表示の切り替え時（日本語 ⇆ English）、ウィンドウサイズが意図せず極端に縮小してしまう致命的なUIバグを修正する。
- **ターゲットコード**: `src/aigis/ui/MainFrame.java`, `src/aigis/i18n/I18n.java`
- **アプローチ**: 
  `rebuildUI()` 内でコンポーネントが再配置（`pack()` や `validate()`）される際、現在のウィンドウサイズ（`getSize()`）や位置（`getLocation()`）を一時退避・再設定するロジックを組み込み、サイズを強制維持させる。

### 2. 【3Dグラフィックス拡張Agent】（Skill: 複数モデル制御・ビューポート分割）
- **ミッション**: 現在、同じモデル同士（イトカワのみ）で最大4分割表示している機能を、異なる小惑星モデル（イトカワとリュウグウ）を左右に並べて比較表示できるように拡張する。
- **ターゲットコード**: `MainFrame.java` 内の `private GLSplitWindow window;` および複数表示（Multi-View）制御クラス。
- **アプローチ**: 分割されたビューポート（インデックス）ごとに、異なる `.obj` ファイル（`Itokawa_49152.obj` や `Ryugu` 等）のインスタンスをロード・保持できる独立したデータ構造へ書き換える。

### 3. 【教育機能・数学ロジックAgent】（Skill: 2D/3D座標変換・リアルタイム描画）
- **ミッション**: カメラのズーム（FOVやDistance）に連動して、リアルタイムに縮尺が「0〜800m」のように変化する「2Dスケールバー」を画面隅にオーバーレイ表示する。
- **ターゲットコード**: `MainFrame.java` 内の `Distance` や `FOV` の変数、3D描画レンダリングループ、または前面描画用JLayeredPane。
- **アプローチ**: 3D空間上の実際の距離から、画面上の2Dピクセル比率へと変換する計算ロジック（ $3D座標 \to 2D画面比率$ ）を実装し、動的にラインと数値を描画する。

## ■ 共通開発プロセス
1. **タスク読込**: `tasks/` ディレクトリ内の指定されたタスクを読み込む。
2. **バックアップ**: 【最重要】変更対象のフォルダを `src/backups_disabled/`（ビルド対象外）へコピーして100%保存する。`src/src/` 配下にバックアップを残さない。
3. **探索と修正**: `messages_ja.properties` などのリソース構造を破壊しないよう慎重に修正する。
4. **ビルド確認**: 修正後、コンパイルが正常に通るか確認し、`git diff` と共に報告する。
